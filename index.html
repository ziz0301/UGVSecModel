UGV Security Modelling Based on AVHARM

We built a security modelling for UGV based on the AVHARM model introduced in the paper: “Graphical Security Modelling for Autonomous Vehicles: A Novel Approach to Threat Analysis and Defense Evaluation.”

The original AVHARM framework models in-vehicle networks using: Attack Graphs, Attack Fault Trees, and Risk-based Scoring.

We adapted this model to UGV architecture using a sample scenario, detailing ECUs, buses, and vulnerabilities. We assume a high-threat scenario where all critical nodes are vulnerable to root compromise.

System Architecture

We design an intensive cyber attack scenario where all nodes contain vulnerabilities that allow an adversary to gain root access to each component's system. There are two types of nodes in the architecture: Component Nodes and Access Nodes.

  • Component Nodes: Responsible for main processing and control tasks of the UGV.
  • Access Nodes: Represent peripheral systems that either supply information to or receive commands from component nodes.

Component Nodes

  1. Autonomous Processor (AP)
    • AP_sensor: Processes data from sensors (camera, radar, LIDAR).
    • AP_nav: Handles navigation data from GNSS.
    • AP_control: Converts data into control commands for the Vehicle Controller.
  2. Vehicle Controller (VC)
    • VC_rcv: Receives control commands from AP or others.
    • VC_pass: Sends commands to steering, drive, brake systems.
    • VC_start: Starts the vehicle; can issue manual drive commands.
  3. Wire/Wireless Link (WL)
    • WL_recon: Receives commands from the external control station.
    • WL_trans: Forwards commands to vehicle or payload systems.
  4. Payload Module (PM)
    • PM_comm: Communicates with WL for commands.
    • PM_rcv: Receives targeting data from EO/IR.
  5. Control Station (CS)
    • CS_auth: Authenticates users for remote control via WL.

Access Nodes

Connected to Autonomous Processor (AP):

  • Camera: via Ethernet
  • Radar: via Ethernet
  • LIDAR: via Ethernet
  • NavSys: via GNSS

Connected to Vehicle Controller (VC):

  • Steering System: via CCAN
  • Driving System: via CCAN
  • Braking System: via CCAN
  • PowerDist: via BCAN
  • BSA: Battery Safety Actuator

Connected to Remote-Controlled Payload Module (PM):

  • Payload: via Ethernet
  • EO-IR: via Ethernet
